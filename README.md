# Deribit Price Tracker API

Асинхронный сервис на FastAPI для мониторинга курсов криптовалют (BTC/USD, ETH/USD) с биржи Deribit. Проект включает в себя автоматизированный сбор данных, сохранение в PostgreSQL и предоставление REST API для анализа истории цен.

## 🚀 Основной стек технологий

* **Python 3.13** (использование последних возможностей языка)
* **FastAPI** — высокопроизводительный веб-фреймворк.
* **SQLAlchemy 2.0 (Async)** — работа с базой данных в асинхронном режиме.
* **Celery + Redis** — планировщик и обработчик фоновых задач.
* **uv** — современный и сверхбыстрый менеджер пакетов (вместо pip/poetry).
* **Docker / Docker Compose** — полная контейнеризация системы.

## 🛠 Design Decisions (Архитектурные решения)

При разработке проекта были приняты следующие ключевые решения:

1. **Асинхронная архитектура**: Весь путь запроса (FastAPI -> SQLAlchemy -> Asyncpg) реализован асинхронно. Это позволяет эффективно обрабатывать большое количество одновременных соединений при минимальном потреблении ресурсов.
2. **Использование `uv**`: Выбор `uv` обусловлен его скоростью и надежностью при сборке Docker-образов. Использование `uv.lock` гарантирует идентичность окружения на всех этапах разработки.
3. **Разделение ответственности (Separation of Concerns)**:
* Фоновые задачи (Celery Worker/Beat) изолированы от основного API. Это гарантирует, что даже при высокой нагрузке на сбор данных или временной недоступности биржи, API будет отвечать стабильно.
* Бизнес-логика вынесена в слой `services`, оставляя роуты максимально легкими.


4. **Безопасность (Security)**:
* Приложение в Docker запускается от имени не-привилегированного пользователя (`appuser`).
* Использованы настройки `no-new-privileges` и ограничены `capabilities` ядра в Docker Compose.


5. **UUID как первичные ключи**: В качестве `id` для записей цен используются UUIDv4. Это защищает от атак перебором (ID enumeration) и упрощает масштабирование БД.
6. **Выбор Redis как брокера**: Redis выбран за минимальные задержки (low latency), что критично для обработки финансовых данных в реальном времени.

## 📦 Развертывание

### 1. Подготовка окружения

Создайте файл `.env` в корневой директории на основе примера:

```bash
cp .env.example .env

```

*Убедитесь, что параметры подключения к БД в `.env` совпадают с переменными в docker-compose.*

### 2. Запуск через Docker Compose

Проект полностью автоматизирован. Для сборки и запуска всех сервисов (API, Worker, Beat, Redis, DB) выполните:

```bash
make run
# или
docker-compose up --build -d

```

### 3. Запуск тестов

Тесты запускаются внутри контейнера приложения:

```bash
make test
# или
docker-compose exec app uv run pytest

```

## 📖 Документация API

После запуска API доступно по адресу `http://localhost:8000`. Интерактивная документация:

* **Swagger UI (OpenAPI)**: [http://localhost:8000/docs](https://www.google.com/search?q=http://localhost:8000/docs) — для тестирования запросов.
* **ReDoc**: [http://localhost:8000/redoc](https://www.google.com/search?q=http://localhost:8000/redoc) — для детального изучения схем данных.

**Основные эндпоинты:**

* `GET /prices/latest` — Текущая цена для выбранного тикера (btc/eth).
* `GET /prices/history` — История цен за указанный период (start_ts, end_ts).
* `GET /prices/all` — Все сохраненные данные по тикеру.
